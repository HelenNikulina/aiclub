🎓 ГАЙД: Як безпечно передавати API-ключі своєму боту (і не зливати їх у git)

Будь-який серйозний бот рано чи пізно вимагає API-ключі: OpenAI, Anthropic, Cloudflare, Stripe, Resend, WayForPay, Telegram, Google. Питання — як їх туди завантажити **без витоку**. Найчастіша помилка новачків — вставити ключ у код, закомітити в git, запушити на GitHub. За 10 хвилин боти-скрейпери знаходять цей ключ, і з вашого Stripe списали €500 або з OpenAI вигоріло $1000 кредитів. У публічних репозиторіях це трапляється щоденно.

Цей гайд показує найбезпечніший спосіб: **передавати ключі прямо боту через Telegram-команду** — без git, без терміналу, без копіювання у файл руками.

**Чому цей спосіб найкращий:**

- Ключ ніколи не торкається репозиторію (git history чистий)
- Ключ ніколи не вводиться через термінал (де його видно колегам по плечу)
- Бот автоматично пише ключ у `.env` з правами `chmod 600` (читає тільки власник)
- Бот **видаляє ваше повідомлення з чату** після збереження — ніяких слідів у Telegram-історії
- Все відбувається з телефону за 10 секунд

**Що потрібно мати**

Власного Telegram-бота, який ви вже запустили на сервері (як у гайді №X «Telegram-бот як повноцінний агент»). У його `.env` лежать всі ключі, які він читає при старті.

**Крок 1 — додаємо команду `/secret` в код бота**

Знайдіть місце обробки команд (зазвичай функція типу `handle_command` або `dispatcher`). Додайте новий блок:

```python
elif cmd == '/secret':
    # /secret KEY=VALUE — безпечне збереження ключа в .env
    rest = text[len('/secret'):].strip()
    if '=' not in rest:
        send(chat_id, '❌ Формат: /secret KEY=VALUE')
        return
    k, v = rest.split('=', 1)
    k = k.strip(); v = v.strip()
    if not k or not v:
        send(chat_id, '❌ Порожній ключ або значення.')
        return
    env_p = '/home/botuser/claudebot/.env'  # шлях до вашого .env
    lines = open(env_p).readlines() if os.path.exists(env_p) else []
    # Видаляємо стару версію цього ключа якщо була
    lines = [l for l in lines if not l.startswith(k + '=')]
    lines.append(f'{k}={v}\n')
    open(env_p, 'w').writelines(lines)
    os.chmod(env_p, 0o600)  # тільки власник читає
    send(chat_id, f'✅ {k} збережено в .env ({len(v)} символів). Поза git, безпечно.')
    # Видаляємо повідомлення з ключем з Telegram
    try:
        tg('deleteMessage', chat_id=chat_id, message_id=msg.get('message_id'))
    except Exception:
        pass
```

**ВАЖЛИВО:** обовʼязково закрийте цю команду по `user_id` — тільки ви маєте могти її викликати. Інакше будь-хто, хто додасть бота в чат, зможе перезаписувати ваші ключі. Перевірка робиться через `ALLOWED_IDS` — список Telegram ID власників.

**Крок 2 — перезапускаєте бота**

Бот тепер розуміє `/secret`.

**Крок 3 — надсилаєте ключ ботові**

У чат з ботом одним повідомленням:

```
/secret OPENAI_API_KEY=sk-proj-abc123...
```

Через 1-2 секунди:
- Ваше повідомлення з ключем — **зникає з чату** (бот видалив)
- Зʼявляється відповідь «✅ OPENAI_API_KEY збережено в .env (164 символів). Поза git, безпечно.»
- Ключ лежить на сервері у `.env`, чекає що його прочитає бот при наступному перезапуску

**Крок 4 — рестарт сервіса, щоб бот підхопив новий ключ**

Якщо бот запускається через systemd:

```python
# В /secret після збереження додайте:
subprocess.run(['systemctl', 'restart', 'mybot'])
```

Або робите окрему команду `/restart`, яка робить це за вашою вказівкою.

**Чого НЕ робити (поширені помилки)**

- НЕ комітьте `.env` у git. У `.gitignore` мають бути рядки `.env`, `.env.local`, `*.key`, `*.pem`.
- НЕ передавайте ключ через SSH/термінал у public Wi-Fi. Якщо все ж довелося — змініть ключ потім.
- НЕ зберігайте всі ключі в одному акаунті. Розділяйте: dev-ключі окремо, prod окремо, з різними лімітами.
- НЕ зберігайте ключі в історії bash (`~/.bash_history`). Якщо набирали `export KEY=...` в терміналі — почистіть історію командою `history -c && history -w`.
- НЕ давайте Claude Code/будь-якому AI-агенту ключ прямо в чат. Чат-сесії індексуються, логуються, можуть лежати у бекапах. Використовуйте `/secret`.

**Що робити, якщо ключ випадково потрапив у репозиторій**

1. Зайдіть на сайт сервісу (Stripe / OpenAI / Cloudflare) → API Tokens → **Revoke** старий ключ. Це найважливіше — за 5 хвилин.
2. Створіть новий.
3. Надішліть новий через `/secret`.
4. (Опційно) Очистіть git-історію через `git filter-repo` або BFG Repo-Cleaner. Але це довго і часто не потрібно — головне revoke зробити швидко.

**Бонус: ротація ключів раз на 30-90 днів**

Налаштуйте крон-нагадування в боті: раз на 30 днів він пише вам «час обновити OPENAI_API_KEY» і чекає поки ви надішлете нову версію через `/secret`. Це 5 хвилин дисципліни на квартал, але рятує від ситуації коли «півроку тому залив ключ і він всюди валяється».

**Підсумок**

Команда `/secret KEY=VALUE` в Telegram-боті — це найшвидший і найбезпечніший спосіб завантажити будь-який API-ключ на ваш сервер. Один раз додаєте 20 рядків коду — і всі майбутні ключі ходять цим каналом. Git залишається чистим, термінал нікому не показуєте, історія Telegram автоматично затирається.
