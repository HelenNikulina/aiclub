🎓 ГАЙД №16: Дай своєму AI-агенту справжній браузер (коли код не може відкрити сайт)

Знайома ситуація? Просиш агента зайти на сайт, спарсити дані, залогінитись кудись — а він відповідає: «не маю доступу», «403», «Access Denied», «заблоковано». Це не агент поганий. Це його пісочниця (або VPS) сайт сприймає як бота й не пускає.

Рішення просте за ідеєю: **хай агент працює не зі свого сервера, а з ТВОГО справжнього браузера на твоєму комп'ютері.** Для сайту це жива людина — ти — а не робот.

⸻

🧠 ЯК ЦЕ ПРАЦЮЄ (без магії)

Агент не підключається напряму до твого комп'ютера (це майже завжди заблоковано). Він спілкується через GitHub — як через поштову скриньку:

1. Агент кладе завдання у файл у твоєму репозиторії
2. Маленька програма на твоєму ноуті раз на 5 сек дивиться GitHub
3. Бачить завдання → відкриває твій Chrome → робить що треба
4. Кладе результат назад у GitHub → агент його читає

Той самий принцип, що й «командний міст» до сервера, тільки з браузером.

⸻

⚙️ ВСТАНОВЛЕННЯ (один раз, ~10 хвилин)

📍 Крок 1 — Python і бібліотеки

На Mac (Termiнal), вставити одним блоком:
```
python3 -m venv ~/claude-relay && source ~/claude-relay/bin/activate && pip install playwright requests && python -m playwright install chromium
```
Якщо вискочить вікно «Install command line developer tools» — натисни Install, зачекай, повтори.

На Windows (PowerShell):
```
py -m venv $HOME\claude-relay; & $HOME\claude-relay\Scripts\Activate.ps1; pip install playwright requests; python -m playwright install chromium
```

📍 Крок 2 — ключ GitHub

Відкрий https://github.com/settings/tokens?type=beta
• Repository access → тільки твій репозиторій (той, з яким працює агент)
• Permissions → Contents → Read and write
• Згенеруй і скопіюй токен (показується один раз!)

📍 Крок 3 — запуск

Візьми файл `browser_relay.py` (його дасть твій агент) і запусти, підставивши свої дані:
```
export GITHUB_TOKEN="твій_токен"
export RELAY_REPO="твій-логін/твій-репозиторій"
export RELAY_BRANCH="гілка-яку-слухає-агент"
python browser_relay.py
```

Відкриється вікно Chrome — це «браузер агента». На сайтах з логіном залогінься в ньому один раз, далі він пам'ятає.

⸻

🚀 ЯК КОРИСТУВАТИСЬ

Просто залиш вікно і програму відкритими, поки потрібен браузер. Кажеш агенту: «відкрий цей сайт / збери дані звідси» — і він робить це у твоєму Chrome. Закрила вікно — зв'язок зник, нічого не ламається. Знову треба — знову запускаєш.

⸻

⚠️ ЧЕСНО ПРО ОБМЕЖЕННЯ

• Це не фоновий сервіс — працює лише поки програма запущена. Для разових задач це нормально.
• Дуже захищені сайти (Akamai/DataDome — наприклад LoopNet, Crexi) можуть блокувати навіть справжній браузер під автоматизацією. Більшість звичайних сайтів — працюють.
• Програма виконує код, який агент кладе в чергу, на твоєму комп'ютері — та сама довіра, що й до командного містка до сервера. Токен тримай лише в змінній оточення, не світи в чатах.

⸻

💬 ПІДСУМОК

Якщо твій агент «не може відкрити сайт через безпеку» — тепер ти знаєш, що робити: дай йому свій справжній браузер. Один раз налаштувала — і ця проблема закрита назавжди.
