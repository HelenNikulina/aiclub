# Гайд: Пам'ять Claude Code між сесіями (для claude.ai/code)

Цей гайд — для тих, хто працює з Claude Code через браузер або десктоп-додаток (`claude.ai/code`). Якщо ти користуєшся **локальним Claude Code CLI** в терміналі — підхід інший, дивись розділ «Локальний CLI» в кінці.

---

## Що отримаємо

- Claude **знає тебе і твої проєкти** в кожному новому чаті
- Не треба щоразу пояснювати «я Python-розробник», «використовую FastAPI», «мій бот в гілці X»
- Claude **сам записує** важливі рішення і факти по ходу роботи — щоб не забути після падіння сесії

---

## Чому НЕ працює «стандартний» підхід

Тобі могли порадити налаштувати `~/.claude/CLAUDE.md`, `~/.claude/MEMORY.md` або хуки в `~/.claude/settings.json`. **Для `claude.ai/code` це безглуздо.** Чому:

- Кожен новий чат запускається в **новому хмарному сендбоксі** (тимчасовий контейнер).
- У сендбоксі **порожня домашня папка** `~/.claude/`. Все, що ти туди записала в попередньому чаті, зникло.
- Хуки в `settings.json` не встигають навіть раз спрацювати — файл не існує в новому чаті.

Те, що **переживає** новий сендбокс — це **твій GitHub-репозиторій**, бо він клонується з GitHub при старті кожного чату. Висновок: **пам'ять має лежати в репо**, не в домашній папці.

---

## Робочий підхід — два файли в репо

| Файл | Шлях | Що містить | Хто пише |
|---|---|---|---|
| `CLAUDE.md` | корінь репо | Постійний контекст: хто ти, правила, інфраструктура | Ти + Claude по запиту |
| `MEMORY.md` | корінь репо | Саммарі сесій: рішення, зміни, незавершене | Claude — автоматично, проактивно |

Обидва файли коммітяться в гілку **`main`**. Кожен новий чат клонує `main` → бачить ці файли → знає тебе.

---

## Покроково

### Крок 1. Відкрий свій робочий репо в Claude Code

Зайди на `https://claude.ai/code`, відкрий новий чат на тому репо, з яким ти регулярно працюєш. Якщо такого нема — створи в GitHub новий репо (наприклад, `my-claude-memory`) і відкрий його.

### Крок 2. Переконайся, що default branch — `main`

Це критично. Якщо default інший — новий чат клонує його і пам'ять не побачить.

1. Відкрий `https://github.com/<юзернейм>/<репо>/settings`
2. Зліва: `General` → проскроль до низу
3. Знайди блок **Default branch**
4. Якщо там не `main` — натисни ⇄ → вибери `main` → **Update** → підтверди

### Крок 3. Попроси Claude створити `CLAUDE.md`

Скопіюй цей запит у чат:

> Створи в корені репо файл `CLAUDE.md` для постійного контексту. Структура:
> - Заголовок «Контекст для Claude — прочитай перед будь-якою дією»
> - Розділ «Пам'ять сесій»: «також одразу прочитай `MEMORY.md` у корені репо»
> - Розділ «Хто я» (порожній, я заповню)
> - Розділ «Правила роботи зі мною» (порожній)
> - Розділ «Мої проєкти» (порожній)
>
> Закомить у гілку `main` і запушай через `git push origin main`.

### Крок 4. Попроси Claude створити `MEMORY.md`

> Створи в корені репо файл `MEMORY.md` із заголовком «Пам'ять — саммарі сесій» і коротким описом формату: нові секції в кінець файлу, заголовок виду `## YYYY-MM-DD HH:MM — короткий опис`, 3-7 буллетів. Закомить у `main`, запушай.

### Крок 5. Заповни `CLAUDE.md` своїм контекстом

Просто розкажи Claude про себе. Приклад:

> Запиши в `CLAUDE.md` (розділ «Хто я» і «Правила»):
> - Я Python-розробник початкового рівня, потрібні детальні пояснення з прикладами
> - Завжди відповідай українською
> - Мій основний проєкт — телеграм-бот, гілка `bot-main`
> - Сервер на Hostinger VPS, доступ через Browser terminal
>
> Закомить у `main`, запушай.

### Крок 6. Додай правило проактивності в `CLAUDE.md`

Без цього Claude писатиме в `MEMORY.md` тільки коли ти попросиш. З цим — сам.

> Допиши в `CLAUDE.md` розділ «Як оновлювати MEMORY.md»:
>
> Не чекати команди «онови пам'ять» і не чекати кінця сесії. Записувати одразу при:
> - прийнятому технічному рішенні
> - значимому коміті/пуші
> - виявленому факті/обмеженні, яке знадобиться далі
> - незавершеній задачі, яку треба передати наступній сесії
>
> Кожне оновлення `MEMORY.md` робити **тільки в гілці `main`**:
> ```
> current=$(git branch --show-current)
> git checkout main && git pull origin main
> # Edit MEMORY.md (нова секція в кінець)
> git add MEMORY.md && git commit -m "memory: <опис>" && git push origin main
> git checkout "$current"
> ```
>
> Закомить це доповнення в `main`, запушай.

### Крок 7. Перевір у новому чаті

1. Закрий поточний чат
2. Відкрий той самий репо в **новому** чаті на `claude.ai/code`
3. Спитай: **«Що ти про мене знаєш?»**
4. Має одразу переказати все з `CLAUDE.md` і `MEMORY.md`

Якщо переказує — все працює. Якщо «не пам'ятаю нічого» — див. таблицю нижче.

---

## Що НЕ писати в ці файли

`CLAUDE.md` і `MEMORY.md` лежать у git → видно всім, хто має доступ до репо. Якщо репо публічний — взагалі всім в інтернеті.

**Заборонено:**

- API-ключі, токени, паролі (навіть тестові — звичка)
- Особисті дані третіх людей
- Production-секрети — їм місце в `.env` + `.gitignore`

Для приватного репо тестові токени технічно можна, але звичка тримати все в `.env` надійніша.

---

## Як чистити, коли `MEMORY.md` розпухне

Раз на місяць:

> Відкрий `MEMORY.md` на гілці `main`, прибери секції старші за місяць (крім незавершених задач і важливих рішень). Закомить, запушай у `main`.

---

## Що лагодити, якщо не працює

| Симптом | Причина | Рішення |
|---|---|---|
| Новий чат каже «нічого не пам'ятаю» | Default branch ≠ `main` | GitHub → Settings → General → Default branch → `main` |
| `MEMORY.md` не оновлюється сам | Claude не запам'ятав правило проактивності | «Прочитай `CLAUDE.md` повністю і дотримуйся розділу про MEMORY.md» |
| Помилка `git checkout main` | На робочій гілці незакомічені зміни | Спочатку `git add . && git commit -m "wip"`, потім `checkout main` |
| Коміти в `MEMORY.md` йдуть, але новий чат не бачить | Запушено у feature-гілку, не в `main` | `git log origin/main -- MEMORY.md` — перевір, що твої коміти там |

## Локальний CLI — окремий випадок

Якщо ти використовуєш Claude Code **у терміналі на своєму комп'ютері** (`claude` команда, не через браузер):

- Домашня папка `~/.claude/` **не стирається** між сесіями
- Можна налаштувати **хуки** для автоматичного саммарі через Stop hook
- Підхід з `~/.claude/MEMORY.md` + `settings.json` працює

Але це окремий світ. Більшість сьогодні через `claude.ai/code` — для них працює тільки репо-based підхід з цього гайду.

---

## Головні правила

1. **Пам'ять живе в репо** на гілці `main`, не в `~/.claude/`. У сендбоксі `~/.claude/` стирається.
2. **Default branch має бути `main`** — інакше нові чати клонують іншу гілку.
3. **`CLAUDE.md`** — постійне (хто ти, правила). **`MEMORY.md`** — динаміка (що було в сесіях).
4. **Оновлюй `MEMORY.md` тільки в `main`**. Feature-гілки не видно новим чатам.
5. **Жодних секретів у git.** Токени — в `.env`.
6. **Правило проактивності** в `CLAUDE.md` важливіше за нагадування «онови пам'ять» — сесія може впасти раніше, ніж ти про це згадаєш.
