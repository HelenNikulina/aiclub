🎓 ГАЙД: Запускаємо рекламу у Facebook через Claude Code — від токена до щоденного звіту

Цей гайд для тих, хто вже пройшов попередні гайди каналу: у тебе є VPS з Claude Code, Telegram-бот підключений до сервера, cmd_runner працює. Якщо ні — спочатку гайди з налаштування VPS і Claude Code.

**Що робиш ти:** даєш Claude Code один токен і опис продукту.
**Що робить Claude Code:** створює піксель, налаштовує кампанію, пише скрипт аналітики, ставить cron — сам, через промти нижче.

---

## Що потрібно від тебе (5 хвилин у Facebook)

Одна дія вручну — створити токен системного користувача. Це технічний акаунт, через який Claude Code керуватиме рекламою.

**Кроки:**
1. Зайди на [business.facebook.com](https://business.facebook.com) → Налаштування → **Системні користувачі**
2. Натисни «Додати» → дай ім'я (наприклад `claude-agent`) → роль «Адміністратор»
3. Натисни «Додати активи» → вибери свій рекламний акаунт → постав усі галочки
4. Натисни «Створити токен» → вибери свій застосунок → постав галочки:
   - `ads_read` `ads_management` `business_management` `pages_read_engagement`
5. Скопіюй токен — довгий рядок, починається з `EAAg...`
6. Скопіюй ID рекламного акаунту — в URL кабінету або в налаштуваннях, формат `act_123456789`

Все. Далі працює Claude Code.

## Промт 1 — Зберегти токени на сервер

Скопіюй цей текст у Claude Code, підстав свої значення:

```
Збережи на сервер у .env файл наступні токени:
META_ACCESS_TOKEN=EAAg...твій токен...
META_AD_ACCOUNT_ID=act_123456789
META_PIXEL_ID= (поки порожньо, створимо на наступному кроці)

Використай cmd_runner щоб додати їх у /home/botuser/claudebot/.env
Перевір що дублів немає.
```

**Що зробить Claude Code:** запише токени в .env на сервері через cmd_runner. У git нічого не потрапить.

## Промт 2 — Створити піксель через API

```
Створи Meta Pixel для мого сайту через Marketing API.
Назва пікселя: [назва твого проєкту]
Сайт: [твій домен, наприклад getsoulrise.com]

Після створення:
1. Збережи Pixel ID у .env як META_PIXEL_ID
2. Дай мені готовий HTML-код для вставки в <head> сайту
3. Дай код події Purchase — для вставки на сторінку після оплати
```

**Що зробить Claude Code:** викличе API `POST /act_{id}/adspixels`, отримає Pixel ID, збереже в .env, віддасть два готові HTML-сніппети для вставки на сайт.

**Що робиш ти:** вставляєш два шматки HTML на сайт (у Lovable, WordPress або через розробника).

## Промт 4 — Створити весь контент і запустити кампанію

Це головний промт. Claude Code сам напише тексти, згенерує зображення через DALL-E, завантажить їх у Meta і створить всю структуру кампанії — тобі тільки підтвердити запуск.

> Для генерації зображень потрібен OpenAI API ключ. Якщо немає — додай спочатку: «Збережи OPENAI_API_KEY=sk-... у .env на сервері»

```
Створи рекламну кампанію у Facebook через Marketing API.

Продукт: [назва і суть — наприклад "онлайн-курс з техніки голосу для жінок"]
Ціна продукту: [наприклад $97]
Сайт (лендінг): [посилання]
Бюджет на день: [наприклад $20]
Аудиторія: [країна, вік, стать — наприклад "жінки, США, 25-65 років"]
Ключовий біль клієнта: [наприклад "боїться говорити публічно, голос тремтить, губиться у словах"]
Стиль візуалу: [наприклад "реальна жінка, впевнена поза, виступає перед аудиторією, теплі тони"]

Вимоги:
- Мета кампанії: OUTCOME_SALES (конверсії, не кліки!)
- Оптимізація ad set: OFFSITE_CONVERSIONS → Purchase
- Використати Pixel ID з .env

Зроби сам — все по порядку:
1. Напиши 3 варіанти тексту оголошення (формула PAS: біль → посилення → рішення)
2. Напиши 3 варіанти заголовку (хуки: питання / проблема / контраст)
3. Згенеруй 3 зображення через DALL-E API (1024×1024, формат для Facebook Feed)
   — кожне під відповідний заголовок
4. Завантаж зображення в рекламний акаунт Meta через API (/adimages)
5. Створи кампанію, ad set і 3 варіанти creative з різними зв'язками текст+картинка
6. Статус кампанії: PAUSED
7. Покажи мені всі 3 варіанти оголошень (текст + опис картинки)
8. Чекай мого підтвердження. Після "запускай" — переключи всі на ACTIVE
```

**Що зробить Claude Code:**
- Напише рекламні тексти і заголовки
- Згенерує зображення через DALL-E 3 API
- Завантажить їх у Meta через `/act_{id}/adimages`
- Створить 3 готові оголошення (A/B тест з першого дня)
- Покаже тобі на перевірку і чекатиме «запускай»

**Що робиш ти:** пишеш «запускай» — і все.

> ⚠️ Claude тримає все на паузі до твого підтвердження. Фінансове рішення залишається за тобою.

## Промт 5 — Налаштувати щоденний звіт у Telegram

```
Напиши скрипт щоденної аналітики по моїх Facebook-кампаніях і задеплой його на сервер.

Скрипт повинен:
- Брати токени з /home/botuser/claudebot/.env
- Запитувати Meta API: дані за вчора, сьогодні і з початку запуску кампаній
- Рахувати для кожної кампанії: витрати, кліки, покази, CTR, CPC, CPM, покупки, дохід, CPA, ROAS
- Показувати кольорові маркери: 🟢 є покупки / 🟡 CTR > 2.5% / 🔴 слабкий
- Наприкінці додавати блок "Коротко по суті": лідер, аутсайдер, загальний ROAS і висновок
- Надсилати у Telegram через бота на мій OWNER_CHAT_ID
- Мова звіту: українська

Після деплою постав cron на 06:00 UTC (09:00 за Києвом) щодня.
Одразу запусти тест — я повинна отримати повідомлення просто зараз.
```

**Що зробить Claude Code:** напише Python-скрипт, задеплоїть на VPS через cmd_runner (base64-кодування, без коміту токенів у git), поставить cron, надішле тестове повідомлення в Telegram.

## Промт 7 — Налаштувати CAPI (серверні події, відновить втрати від iOS)

Після перших покупок по рекламі:

```
Налаштуй Conversions API (CAPI) — серверну передачу подій покупки в Meta.
У мене Stripe як платіжна система.

Мені потрібно:
1. Створити capi_server.py — Flask-сервер на порту 8182, приймає Stripe webhook
2. При події checkout.session.completed → надіслати Purchase в Meta через CAPI
3. Дедуплікація з Pixel через event_id = session['id']
4. Email покупця хешувати через sha256 перед відправкою
5. Задеплоїти на сервер

Токени STRIPE_SECRET_KEY і META_CAPI_TOKEN вже в .env.
Після деплою скажи мені URL для вставки в Stripe Webhooks.
```

**Навіщо це:** iOS/AdBlocker блокують ~40% подій від Pixel. CAPI надсилає ті самі події з сервера — Meta бачить у 1.5–2 рази більше покупок → алгоритм краще навчається → ROAS зростає.
