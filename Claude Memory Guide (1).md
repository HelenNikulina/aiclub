# Гайд: прокачай пам'ять свого Claude Code

Цей гайд навчить тебе зробити так, щоб Claude **пам'ятав тебе і твої проєкти** між різними сесіями — і не доводилося щоразу пояснювати заново «я frontend-розробник», «використовую TypeScript», «мій проєкт у папці X».

Є два рівні. **Рівень 1** працює у всіх (GUI, десктоп, термінал — без різниці). **Рівень 2** — для тих, хто готовий поколупати один файл налаштувань.

---

## Навіщо це потрібно

За замовчуванням Claude Code забуває розмову після кожної сесії. Але він уміє читати **спеціальні файли пам'яті** автоматично на початку кожної нової сесії.

Файлів два, у них різні ролі:

| Файл | Роль | Хто заповнює |
|---|---|---|
| `CLAUDE.md` | Постійна важлива інформація про тебе і проєкти | Ти або Claude на твоє прохання |
| `MEMORY.md` | Коротке саммарі кожної минулої сесії | Хук (автоматично — Рівень 2) |

---

# Рівень 1 — базова пам'ять (всім підходить)

## Крок 1. Попроси Claude створити тобі файл пам'яті

Відкрий Claude Code і просто напиши:

> Створи файл `~/.claude/CLAUDE.md` для постійної пам'яті. Всередині напиши інструкцію для самого себе: «Коли помічаєш важливу постійну інформацію про користувача (вподобання, проєкти, правила роботи) — одразу додавай її в цей файл, не чекай, коли попросять. Токени і паролі — тільки за явним "запам'ятай"». Далі зроби порожні розділи: Користувач, Уподобання, Проєкти, Угоди і правила.

Claude створить файл сам. Цей файл підвантажується **в кожну нову сесію автоматично** — жодних налаштувань більше не треба.

## Крок 2. Заповнюй пам'ять по ходу розмови

Просто кажи Claude важливі речі як зазвичай — він сам зрозуміє, що записати:

> «Я навчаюся на Python-розробника, мені потрібні детальні пояснення з прикладами»

> «Мій основний проєкт — телеграм-бот, лежить у папці `~/projects/my-bot`»

> «Завжди відповідай мені українською, навіть якщо код англійською»

Claude має відповісти «запам'ятав: …» і дописати в `CLAUDE.md`.

## Крок 3. Токени і ключі — окремо

Якщо хочеш, щоб Claude пам'ятав твій API-ключ або токен бота — обов'язково скажи слово **«запам'ятай»**:

> «**Запам'ятай** мій токен для тестового бота: 1234:abcdef...»

Без слова «запам'ятай» Claude використовує токен тільки в поточному завданні і НЕ запише у файл (це захист від випадкових витоків).

**Важливо:**

- У `~/.claude/CLAUDE.md` (домашня папка) — ОК для тестових токенів
- У `CLAUDE.md` всередині папки проєкту — **ніколи** не клади секрети, цей файл часто потрапляє в git
- Production-ключі краще зберігати в `.env`, а не в `CLAUDE.md`

## Крок 4. Перевір, що працює

Закрий Claude Code, відкрий заново. Спитай:

> «Що ти про мене пам'ятаєш?»

Має переповісти все, що записано в `CLAUDE.md`.

## Крок 5. Чистка

З часом `CLAUDE.md` розпухне. Раз на пару тижнів кажи:

> «Відкрий `~/.claude/CLAUDE.md`, прибери застаріле і дублі, залиш тільки актуальне»

# Рівень 2 — автозбереження кожної сесії

**Що дає:** в кінці кожної сесії Claude САМ зробить коротку вижимку (3-7 пунктів) і допише у файл `MEMORY.md`. Жодних «онови пам'ять перед виходом» — все саме.

**Кому підходить:** macOS / Linux. На Windows потрібні інші команди (можна у Claude і спитати).

**Що потрібно:** вміти відкривати текстовий редактор і зберігати файли в потрібну папку.

## Крок 1. Встанови `jq` (якщо ще немає)

Це маленька утиліта для роботи з JSON, потрібна хуку.

- **macOS:** `brew install jq` (в терміналі)
- **Linux:** `sudo apt install jq` або `sudo dnf install jq`

## Крок 2. Створи файл `~/.claude/MEMORY.md`

Найпростіший спосіб — попроси Claude:

> Створи файл `~/.claude/MEMORY.md` із заголовком «# Persistent Memory» і коротким рядком опису.

## Крок 3. Створи скрипт-хук

Попроси Claude:

> Створи файл `~/.claude/memory-stop-hook.sh` з наступним вмістом (точно як написано), і зроби його виконуваним через `chmod +x`:

```bash
#!/bin/bash
# Recursion guard: sub-Claude від `claude -p` теж тригерить Stop hook
[ "$CLAUDE_MEMORY_HOOK" = "1" ] && exit 0

input=$(cat)
session_id=$(echo "$input" | jq -r '.session_id // empty')
[ -z "$session_id" ] && exit 0

transcript=$(find "$HOME/.claude/projects" -name "${session_id}.jsonl" -type f 2>/dev/null | head -1)
[ -z "$transcript" ] || [ ! -f "$transcript" ] && exit 0

mem="$HOME/.claude/MEMORY.md"

content=$(jq -r '
  select(.type=="user" or .type=="assistant")
  | (.message.content // empty)
  | if type == "string" then .
    elif type == "array" then
      map(select(.type=="text") | .text) | join("\n")
    else empty end
' "$transcript" 2>/dev/null | tail -c 50000)

[ -z "$content" ] && exit 0

summary=$(CLAUDE_MEMORY_HOOK=1 timeout 90 claude -p --output-format text \
  "Below is a Claude Code session transcript. Write 3-7 concise bullets capturing: user preferences/style, key decisions, important project facts, anything worth remembering for future sessions. Skip ephemeral chitchat. Output bullets only (use '- '), no preamble, no headings, no code blocks. If nothing memorable, output exactly: NOTHING_TO_REMEMBER

Transcript:
$content" 2>/dev/null)

[ -z "$summary" ] && exit 0
echo "$summary" | grep -q "NOTHING_TO_REMEMBER" && exit 0

{
  printf '\n## %s (session %s)\n%s\n' "$(date '+%Y-%m-%d %H:%M')" "${session_id:0:8}" "$summary"
} >> "$mem"
```

## Крок 4. Підключи хуки в налаштуваннях

Попроси Claude:

> Відкрий `~/.claude/settings.json` (створи, якщо немає). Додай туди два хуки: `SessionStart`, який через `jq` завантажує `~/.claude/MEMORY.md` в контекст як `additionalContext`, і `Stop`, який запускає `~/.claude/memory-stop-hook.sh` асинхронно з таймаутом 120 секунд. Якщо у файлі вже є якісь хуки — НЕ видаляй їх, додай поруч.

Claude зробить це сам. Якщо хочеться перевірити руками — конфіг має виглядати приблизно так:

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "[ -f ~/.claude/MEMORY.md ] && jq -Rs '{hookSpecificOutput: {hookEventName: \"SessionStart\", additionalContext: (\"# Persistent memory loaded from ~/.claude/MEMORY.md\\n\\n\" + .)}}' ~/.claude/MEMORY.md"
          }
        ]
      }
    ],
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/memory-stop-hook.sh",
            "async": true,
            "timeout": 120
          }
        ]
      }
    ]
  }
}
```

## Крок 5. Активуй

Хуки не підхоплюються автоматично. Один раз потрібно:

- Набрати команду `/hooks` в Claude Code (це перезавантажує налаштування), **або**
- Закрити і заново відкрити Claude Code

## Крок 6. Перевір

1. Поговори про щось конкретне
2. Закрий сесію
3. Через 30 секунд відкрий `~/.claude/MEMORY.md` — там має з'явитися нова секція з буллетами
4. Відкрий нову сесію, спитай «що ти пам'ятаєш з минулої сесії?» — має переповісти

## Що лагодити, якщо не працює

| Симптом | Причина | Рішення |
|---|---|---|
| `MEMORY.md` не поповнюється | Хук не підхопився | Відкрий `/hooks` або перезапусти Claude Code |
| У новій сесії не пам'ятає нічого | SessionStart хук не спрацював | Запусти `claude --debug` і подивись логи |
| Скрипт лається на `jq` | `jq` не встановлено | Див. Крок 1 |
| `MEMORY.md` розпух | Накопичилося багато сесій | Чистка вручну або попроси Claude прибрати старе |

# Головні правила

1. **`CLAUDE.md` ≠ `MEMORY.md`.** Перший — постійне і важливе (ти сам). Другий — автоматичне саммарі сесій (хук).
2. **Без «запам'ятай» — не зберігаємо секрети.** Просто згадав токен = Claude використовує, але не пише у файл.
3. **Ніколи не клади секрети в `CLAUDE.md` всередині папки проєкту.** Тільки в глобальний `~/.claude/CLAUDE.md`.
4. **Чисти раз на пару тижнів.** Файли розпухають, контекст забивається.
5. **Якщо щось зламалося** — спитай у самого Claude: «прочитай `~/.claude/settings.json` і скажи, чи все з хуками гаразд».
