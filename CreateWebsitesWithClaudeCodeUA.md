Цей гайд для тих, хто вже пройшов попередні гайди каналу: у тебе є VPS з Claude Code, Telegram-бот підключений до сервера, cmd_runner працює. Якщо ні — спочатку гайди з налаштування VPS і Claude Code.

**Що робиш ти:** даєш Claude Code два токени і опис свого продукту.
**Що робить Claude Code:** генерує код сайту з нуля, деплоїть на Cloudflare Pages, чіпляє домен, перевіряє роботу — сам, через промти нижче.

> Якщо у тебе вже є існуючий сайт (Lovable, Webflow, окремий React-проєкт) — у кінці гайду є опціональний промт «Підключити існуючий код».

## Що потрібно від тебе (5 хвилин)

Дві одноразові дії — далі все автоматично через Claude.

### 1. GitHub-токен

1. Зайди на [github.com/settings/tokens](https://github.com/settings/tokens) → "Generate new token (classic)"
2. Назва: `claude-websites`, scope: ☑ `repo`, без терміну
3. Скопіюй токен — починається з `ghp_...`

### 2. Cloudflare Global API Key

1. Зайди на [dash.cloudflare.com/profile/api-tokens](https://dash.cloudflare.com/profile/api-tokens)
2. Прокрути вниз до секції **API Keys**
3. Напроти **Global API Key** натисни **View** → введи пароль → скопіюй

⚠️ Це майстер-ключ з повним доступом до Cloudflare. Зберігається на сервері з chmod 600 і ніколи не комітиться у git. Якщо засвітиться — на тій самій сторінці кнопка **Roll** генерує новий, старий миттєво відкликається.

Все. Далі працює Claude Code.

## Промт 1 — Зберегти токени на сервер

Скопіюй цей текст у Claude Code, підстав свої значення:

```
Збережи на сервер у /home/botuser/claudebot/.env наступні токени
через cmd_runner (ніколи не коміть у git!):

GITHUB_TOKEN=ghp_твій_токен
CLOUDFLARE_GLOBAL_KEY=cfk_твій_ключ
CLOUDFLARE_EMAIL=твій-email@gmail.com

Через API Cloudflare отримай мій Account ID і теж збережи:
CLOUDFLARE_ACCOUNT_ID=...

Перевір що дублів немає, chmod 600 на .env, поверни мені «success=true»
з verify-ендпоінту обох токенів.
```

**Що зробить Claude Code:** запише токени в `.env` на сервері, через `GET /accounts` отримає твій Account ID, перевірить що ключі робочі, поверне підтвердження.

## Промт 2 — Згенерувати скелет сайту

Тут ти описуєш Claude що тобі потрібно — він створює репо і пише початковий код.

```
Створи мені новий лендінг з нуля.

Продукт: [назва і суть — наприклад "21-денний онлайн-курс для жінок
від хронічних болів у спині"]
Аудиторія: [хто це купує — наприклад "жінки 45-65 років, США"]
Ключовий біль: [наприклад "не може нахилитися без болю, прокидається
скута, страшно що буде гірше з віком"]
Ціна: $19
Кількість уроків: 21
Спікер/експерт: [ім'я і регалії, або "TBD"]
Стиль: [мінімалізм / тепла палітра / медичний / щось інше]

Стек: Vite + React + TypeScript + Tailwind + shadcn/ui (як у moderному
SaaS-стилі). Це той самий стек що у Lovable — щоб у майбутньому міг
імпортувати/експортувати між Lovable і моїм репо.

Зроби сам:
1. Створи приватний репо <назва-проєкту> у моєму GitHub
2. Згенеруй початковий код (npm create vite + конфіг + базові
   shadcn-компоненти + структура папок src/pages, src/components)
3. Створи src/pages/Index.tsx з лендінгом за такою структурою:
   - Hero (заголовок + CTA + соціальний доказ)
   - Symptoms checklist (8-10 конкретних симптомів)
   - Why-this-method-works (4 root causes)
   - About expert (компактна картка з фото-плейсхолдером + регалії)
   - Curriculum (3 тижні × N уроків)
   - Testimonials (3 шт)
   - Pricing ($19)
   - FAQ (5 питань)
   - Final CTA
4. Закоміть і запуш у репо на гілку main
5. Поверни мені посилання на репо
```

**Що зробить Claude Code:** створить репо через GitHub API, згенерує всі файли проєкту через cmd_runner, напише копірайтинг під твою тематику (із застосуванням скіла copywriting якщо встановлено), закоміть і поверне посилання.

**Що робиш ти:** нічого. Чекаєш репо.

## Промт 3 — Деплой на Cloudflare Pages

```
Створи проєкт «<назва-проєкту>» на Cloudflare Pages і задеплой
код з мого репо.

Кроки:
1. POST /accounts/{ACC}/pages/projects з name=<назва>, production_branch=main
2. На сервері: клонуй репо у /home/botuser/build/<назва>/, зроби
   npm install --no-audit, npm run build
3. Додай у dist/_redirects рядок «/* /index.html 200» (для SPA-роутингу)
4. Деплой через npx wrangler@3 pages deploy dist --project-name=<назва>
   (Wrangler latest потребує Node 22+, а на сервері 20 — тому wrangler@3)
5. Поверни мені preview URL виду https://<хеш>.<назва>.pages.dev
6. Перевір curl-ом що / і кілька сторінок повертають 200
```

**Що зробить Claude Code:** створить проєкт на Pages через API, склонує + збере код на сервері, заллє через Wrangler, поверне preview-URL з підтвердженням.

## Промт 4 — Привʼязати власний домен

Якщо домен у тебе вже є на Cloudflare:

```
Привʼяжи мій домен <мій-домен.com> до проєкту Cloudflare Pages «<назва>».

Кроки:
1. POST /accounts/{ACC}/pages/projects/<назва>/domains з name=<мій-домен.com>
2. Знайди zone_id мого домена через GET /zones?name=<мій-домен.com>
3. Якщо у DNS є стара A-запис на чужого хостера — видали її
4. Створи CNAME apex: name=<мій-домен.com>, content=<назва>.pages.dev,
   proxied=true, ttl=Auto
5. Перевір через X-Auth-Email + X-Auth-Key (це Global API Key, не Bearer)
6. Зачекай 30 секунд, потім зроби curl https://<мій-домен.com> і перевір
   що відповідає Cloudflare (server: cloudflare у заголовках)
```

**Що зробить Claude Code:** через Cloudflare API додасть домен у Pages, знайде зону, прибере конфлікти, створить CNAME, дочекається SSL і поверне «✅ домен живий».

Якщо домен ще не на Cloudflare — спочатку додай зону: на dash.cloudflare.com → Add Site → введи домен → перенеси NS-сервери у свого реєстратора (один раз ~10 хвилин).

## Промт 5 — Покращити лендінг за структурою конкурентів

Це головний промт коли треба зробити сторінку конверсійною.

```
Я хочу прокачати свій лендінг src/pages/Index.tsx за прикладом
конкурентів у моїй ніші.

Конкуренти:
- https://<конкурент-1>
- https://<конкурент-2>
- https://<конкурент-3>

Дій так:
1. Витягни сторінки конкурентів через r.jina.ai (вони здебільшого
   за Cloudflare-челенджем, тому curl напряму не працює)
2. Проаналізуй структуру: які блоки, яка довжина курсу, скільки уроків,
   які заголовки, як подають експерта
3. Покращ мою сторінку до того ж рівня:
   - Якщо у них «X-Day Method» — постав таку ж
   - Якщо у них N video lessons — постав N
   - Symptoms checklist — переформулюй під найгостріші болі моєї ЦА
   - Why-method-works — додай root-cause framing
4. Тримай блок «About Expert» компактно — не розтягуй на пів сторінки
5. Зроби лендінг щільним: 5-6 секцій замість 9-10
6. Закоміть на гілці feat/<що-зробив> і деплой preview
7. Поверни preview URL — я подивлюсь і скажу мерджити чи допилити
```

**Що зробить Claude Code:** скрейпить конкурентів через r.jina.ai, складе порівняльну таблицю, перепише сторінку з мерджем секцій, задеплоїть на preview і дасть тобі URL до мерджу в прод.

**Що робиш ти:** дивишся preview, кажеш «ок» або «переробити X». Це єдина точка прийняття дизайнерських рішень.

## Скіли — встановити один раз для якості сайту

Це файли з експертизою. Claude Code автоматично використовує їх коли ти просиш зробити щось пов'язане.

### Скіл 1: web-design-guidelines (рев'ю UI/UX)

```
Встанови скіл web-design-guidelines з:
https://raw.githubusercontent.com/anthropics/skills/main/skills/web-design-guidelines/SKILL.md
у ~/.claude/skills/web-design-guidelines/SKILL.md
```

Що дає: коли просиш «Зроби рев'ю мого UI», Claude перевіряє accessibility, contrast, typography, spacing — за стандартами Apple/Vercel/Stripe.

### Скіл 2: frontend-design (створення дизайну)

```
Встанови скіл frontend-design з vercel-labs/agent-skills.
```

Що дає: Claude генерує лендінги і компоненти у production-grade стилі, без AI-естетики «однакових сайтів».

### Скіл 3: page-cro (conversion optimization)

```
Встанови скіл page-cro для оптимізації конверсії лендінгів.
```

Що дає: Claude автоматично пропонує CRO-покращення (кращі CTA, social proof, скорочення друк-форм, A/B-ідеї) коли працюєш над лендінгом.

### Скіл 4: copywriting (маркетингові тексти)

```
Встанови скіл copywriting для маркетингових текстів.
```

Що дає: коли просиш «перепиши hero, щоб краще продавав» — Claude застосовує формули PAS, AIDA, JTBD замість загальних слів.

## Що ти робиш руками (мінімум)

| Дія | Чому не можна автоматизувати |
|---|---|
| Створити GitHub-токен | Потребує 2FA і кліку в браузері |
| Витягти Cloudflare Global API Key | Потребує паролю від акаунту |
| Описати продукт у промті №2 | Тільки ти знаєш свого клієнта і свій оффер |
| Підтвердити preview перед мерджем у прод | Дизайн — тільки ти суддя |
| Завантажити свої фото/відео | Контент створюєш ти, завантажує Claude |

**Claude Code робить все інше:** створює репо, генерує код сайту з нуля, пише копірайтинг, збирає продакшн-білд, створює проєкт на Cloudflare Pages, чіпляє домен, виписує SSL, перебудовує лендінги за конкурентами, деплоїть нові сторінки, налаштовує аналітику, запускає cron — і все це через cmd_runner на твоєму сервері. У твоєму браузері майже нічого робити не треба.

## Типові граблі і як обходимо

| Граблі | Рішення |
|---|---|
| `api.cloudflare.com` заблокований у sandbox Claude Code | Усі виклики через cmd_runner на сервері — він має повний інтернет |
| Wrangler latest вимагає Node 22+ | Використовуємо wrangler@3 — підтримує Node 20 |
| Конкурент за Cloudflare Bot Challenge — curl бачить «Just a moment...» | Через `r.jina.ai/<URL>` — рендерить JS і повертає чистий markdown |
| `.env` у репо з `pk_live_` Stripe — здається небезпечно | `pk_live_` за дизайном публічні (бандляться у клієнтський JS), безпечно. Небезпечні `sk_live_` і `rk_live_` — їх у репо НЕ повинно бути |
| Денний ліміт деплоїв на free-tier (100/день у Vercel) | Cloudflare Pages — 500 білдів/місяць, безлімітний трафік. Vercel не потрібен |
