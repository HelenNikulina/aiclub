🎓 ГАЙД: Підключаємо весь Google до Claude — Docs, Sheets, Slides, Forms, Gmail

Виявилося, що для багатьох навіть робота з документами — це складно. Цей гайд закриває питання повністю: після нього Claude сам створює таблиці, презентації, опитування, рахує цифри і пише документи — а ти лише кажеш, що потрібно.

**Що стане можливим:**
- «Claude, порахуй у таблиці витрати за місяць і підсвіти, де перевитрата» → готова Google Таблиця
- «Зроби презентацію з цих цифр» → готовий Google Slides
- «Створи опитування для клієнтів із 5 питань» → готова Google Форма з посиланням
- «Знайди в пошті всі рахунки за листопад і зведи суми» → таблиця з підсумками
- «Візьми цей шаблон договору і підстав дані клієнта» → готовий документ

## Три рівні підключення — обери свій

Не всім потрібен максимум. Ось три рівні, від найпростішого до найпотужнішого.

| Рівень | Кому | Що вміє | Складність |
|---|---|---|---|
| **1. Вбудований конектор** | усім | читати Drive, Gmail, Календар; шукати по файлах | 2 кліки |
| **2. Google Workspace MCP** | хто хоче, щоб Claude РЕДАГУВАВ | + створювати/правити Sheets, Slides, Forms, Docs, Tasks, Контакти | 15 хвилин |
| **3. Service Account** | для автоматизації на сервері без участі людини | те саме, але працює само за розкладом | просунутий |

Більшості потрібен **Рівень 2** — це золота середина. Розберемо його по кроках.

---

## РІВЕНЬ 1 — Вбудований конектор (2 кліки, для читання)

Найшвидший старт. Працює в Claude Desktop і на claude.ai.

1. Відкрий Claude → **Settings (Налаштування)** → **Connectors (Конектори)**
2. Знайди **Google Drive** і **Google Calendar** → натисни **Connect**
3. Увійди у свій Google-акаунт, дозволь доступ

Готово. Тепер можна: «Знайди мій файл про бюджет», «Що в мене в календарі завтра», «Прочитай останній документ із папки Клієнти».

⚠️ Цей рівень лише **читає**. Створювати і редагувати документи він не вміє — для цього потрібен Рівень 2.

---

## РІВЕНЬ 2 — Google Workspace MCP (15 хвилин, для повного керування)

Це окремий «місток» (MCP-сервер), який дає Claude 12 сервісів Google і понад 100 інструментів: Gmail, Drive, Календар, Docs, Sheets, Slides, Forms, Tasks, Контакти тощо. Після встановлення Claude вміє не лише читати, а й **створювати та редагувати**.

Ми використовуємо перевірений відкритий сервер `google_workspace_mcp`.

### Крок 1 — Отримати ключ доступу в Google Cloud (5 хвилин, один раз)

Це технічний «перепустка», через яку Claude заходитиме в твій Google. Робиться один раз.

1. Зайди на [console.cloud.google.com](https://console.cloud.google.com)
2. Угорі створи новий проєкт (кнопка з назвою проєкту → **New Project** → ім'я, наприклад `claude-google`)
3. У пошуку вгорі набери **«OAuth consent screen»** → відкрий → обери **External** → заповни лише обов'язкові поля (ім'я застосунку, твій email) → Save
4. У пошуку набери **«Credentials»** → **Create Credentials** → **OAuth client ID** → тип **Desktop app** → Create
5. Скопіюй **Client ID** і **Client Secret** — вони знадобляться в кроці 3
6. У пошуку увімкни потрібні API (на кожній сторінці синя кнопка **Enable**): Gmail API, Google Drive API, Google Docs API, Google Sheets API, Google Slides API, Google Forms API, Google Calendar API, Google Tasks API, People API

### Крок 2 — Встановити сервер (3 хвилини)

Потрібен Python 3.10+ і менеджер `uv`. Якщо `uv` немає — встанови:

```
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Сам сервер встановлювати заздалегідь не потрібно — він запуститься однією командою з конфіга нижче.

### Крок 3 — Підключити до Claude Desktop (2 хвилини)

Відкрий файл конфігурації Claude Desktop:
- **Mac:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Додай туди блок (підстав свої Client ID і Secret із кроку 1):

```json
{
  "mcpServers": {
    "google": {
      "command": "uvx",
      "args": ["workspace-mcp", "--tool-tier", "core"],
      "env": {
        "GOOGLE_OAUTH_CLIENT_ID": "сюди-твій-client-id",
        "GOOGLE_OAUTH_CLIENT_SECRET": "сюди-твій-secret"
      }
    }
  }
}
```

Збережи файл і **повністю перезапусти Claude Desktop**.

### Крок 4 — Перший вхід (2 хвилини)

При першому запиті на кшталт «покажи мої таблиці» Claude відкриє браузер і попросить увійти в Google. Один раз дозволяєш — далі все працює само, без повторних входів.

## 7 рецептів — готові промти

Скопіюй і підстав своє. Працює на Рівні 2.

**1. Таблиця з даних**
> Створи Google Таблицю «Витрати травень», заголовки: Категорія, Сума, Коментар. Заповни такими рядками: [перелічи]. Заголовок зроби жирним на темному фоні.

**2. Презентація з цифр**
> Візьми дані з таблиці [посилання] і зроби Google презентацію на 5 слайдів: титул, динаміка по місяцях, топ-3 категорії, висновки, наступні кроки.

**3. Опитування/форма**
> Створи Google Форму «Зворотний зв'язок» із питаннями: оцінка від 1 до 5 (обов'язкове), що сподобалося (текст), що покращити (текст). Надішли мені посилання.

**4. Листи → таблиця підсумків**
> Знайди в Gmail усі листи зі словом «рахунок» за листопад, витягни суми і зведи в Google Таблицю з підсумком унизу.

**5. Документ із шаблону**
> Візьми мій шаблон договору [посилання], створи копію «Договір — ТОВ Ромашка» і підстав: назву клієнта, дату, суму.

**6. Брифінг перед зустріччю**
> Подивись мій календар на завтра, по кожній зустрічі знайди минулі листи з цією людиною і зроби короткий брифінг у Google Документі.

**7. Задачі з пошти**
> Перегляньте непрочитані листи, і де є пряме прохання щось зробити — створи задачу в Google Tasks з дедлайном.

## РІВЕНЬ 3 — Service Account (для автоматизації без людини)

Якщо потрібно, щоб документи створювалися **самі за розкладом** на сервері (наприклад, щоранку звіт у таблицю) — використовується Service Account: технічний акаунт без пароля, з ключем-файлом. Це шлях для тих, у кого є свій сервер/VPS і Claude Code на ньому.

Коротко, чим відрізняється від Рівня 2:
- **OAuth (Рівень 2):** Claude діє «від твого імені», потрібен разовий вхід у браузері. Найкраще для особистої роботи.
- **Service Account (Рівень 3):** окремий робот-акаунт, йому дають доступ до конкретної папки (Share → додати email сервіс-акаунта як редактора). Працює без браузера і без людини — ідеально для серверної автоматизації.

Нюанси Service Account, які економлять години (перевірено на практиці):
- **Сервіс-акаунт НЕ може створювати файли** на особистому Gmail — у нього 0 місця на Drive, будь-яке створення падає з помилкою «storage quota exceeded». Це головна пастка.
- **Рішення — два кроки:** файл спочатку створюється від твого імені (вбудований конектор або ти сам у Drive), а сервіс-акаунт уже **наповнює і форматує** його — редагувати чужі файли в розшареній папці він уміє.
- Особистий Gmail / Контакти / Tasks через сервіс-акаунт читати **не можна** (персональні дані) — для них лише OAuth або App Password.
- Google Форми сервіс-акаунт теж створювати не може — лише через OAuth.
- Експорт Docs у PDF: робоче посилання `docs.google.com/document/d/ID/export?format=pdf`.

## Якщо щось не працює

- **«API not enabled / 403»** — не увімкнув потрібний API в Google Cloud (крок 1.6). Відкрий посилання з помилки, натисни Enable, зачекай хвилину.
- **Claude не бачить сервер** — не перезапустив Claude Desktop повністю (вийди із застосунку, не просто закрий вікно).
- **Просить увійти щоразу** — перевір, що Client ID і Secret у конфізі без помилок і пробілів.
- **`uvx: command not found`** — не встановлений `uv` (крок 2).
