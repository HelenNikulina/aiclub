🎓 ГАЙД: Перетворюємо Telegram-бота на повноцінного агента, що робить автоматизації на вашому сервері

Claude Code — це супер, але є нюанс: треба сидіти за терміналом. А що, якщо ви в дорозі, на телефоні, і хочете просто написати «перевір пошту» або «додай нову автоматизацію» — і щоб воно зробилось саме, прямо на вашому сервері? Цей гайд показує, як зробити власного Telegram-бота, який працює як Claude Code: розуміє задачу, сам виконує команди на сервері, читає й редагує файли, запускає скрипти. Тобто ви керуєте сервером з телефону, звичайними повідомленнями.

**Що стане можливим:**
- Пишете боту «зроби щоденний дайджест пошти о 9:00» → він створює скрипт і ставить у cron
- «покажи, скільки місця на сервері» / «перезапусти сервіс X» → виконує і відповідає
- «додай кнопку в бота і задеплой» → редагує код і піднімає оновлення
- Більше не треба відкривати термінал для рутини — бот робить це сам

---

## Чому це працює: агент = модель + інструменти + цикл

Звичайний бот просто пересилає ваш текст у модель і повертає відповідь словами. **Агент** — це коли моделі дають «руки» (інструменти) і дозволяють діяти в циклі:

1. Ви даєте задачу.
2. Модель вирішує, який інструмент викликати (напр. виконати bash-команду).
3. Ваш код виконує інструмент на сервері і повертає моделі результат.
4. Модель дивиться результат і або викликає наступний інструмент, або дає фінальну відповідь.
5. Кроки 2–4 повторюються, поки задача не виконана.

Саме так працює Claude Code всередині. Ми відтворюємо цей цикл у Telegram-боті.

---

## Три інструменти, яких достатньо

Як не дивно, для «повного агента» вистачає трьох інструментів:

- **bash** — виконати будь-яку команду на сервері
- **read_file** — прочитати файл
- **write_file** — записати/перезаписати файл

Маючи bash, агент може майже все: запускати Python-скрипти, ставити пакети, працювати з cron, дивитись логи. read_file/write_file — щоб акуратно редагувати код.

## Код-патерн (Python: Anthropic SDK + python-telegram-bot)

Опис інструментів для моделі:

```python
TOOLS = [
    {"name": "bash", "description": "Виконати bash-команду на сервері",
     "input_schema": {"type": "object",
        "properties": {"command": {"type": "string"}}, "required": ["command"]}},
    {"name": "read_file", "description": "Прочитати файл",
     "input_schema": {"type": "object",
        "properties": {"path": {"type": "string"}}, "required": ["path"]}},
    {"name": "write_file", "description": "Записати файл (повний вміст)",
     "input_schema": {"type": "object",
        "properties": {"path": {"type": "string"}, "content": {"type": "string"}},
        "required": ["path", "content"]}},
]

SYSTEM = """Ти — агент з доступом до мого VPS-сервера.
Якщо задачу можна виконати на сервері — роби це інструментами, не питай зайвого.
Перед зміною файлу спершу прочитай його. Відповідай коротко: що зробив / результат.
⚠️ Руйнівні команди (rm -rf, видалення, reset) — лише з мого явного дозволу: спершу опиши і запитай."""
```

Виконавці інструментів:

```python
import subprocess

def run_bash(cmd):
    r = subprocess.run(cmd, shell=True, capture_output=True, text=True, timeout=45)
    return (r.stdout + r.stderr)[:4000] or "(порожньо)"

def read_file(path):
    return open(path).read()[:4000]

def write_file(path, content):
    open(path, "w").write(content); return "OK"
```

Сам агент-цикл:

```python
import anthropic
client = anthropic.Anthropic(api_key=API_KEY)

def run_agent(task, send):       # send — функція, що шле текст у Telegram
    messages = [{"role": "user", "content": task}]
    for _ in range(20):          # ліміт кроків — захист від нескінченного циклу
        resp = client.messages.create(
            model="claude-sonnet-4-6", max_tokens=4096,
            system=SYSTEM, tools=TOOLS, messages=messages)
        for b in resp.content:
            if b.type == "text" and b.text.strip():
                send(b.text)                       # показуємо проміжні думки
        if resp.stop_reason != "tool_use":
            break                                   # задача завершена
        results = []
        for b in resp.content:
            if b.type != "tool_use":
                continue
            if b.name == "bash":       out = run_bash(b.input["command"])
            elif b.name == "read_file": out = read_file(b.input["path"])
            elif b.name == "write_file": out = write_file(b.input["path"], b.input["content"])
            results.append({"type": "tool_result", "tool_use_id": b.id, "content": out})
        messages.append({"role": "assistant", "content": resp.content})
        messages.append({"role": "user", "content": results})
```

Підключення до Telegram (python-telegram-bot): у хендлері повідомлення викликаєте `run_agent(text, send)`, де `send` — відправка відповіді користувачу. Усе.

## Безпека (обов'язково)

Агент із доступом до сервера — потужно і водночас небезпечно. Мінімум захисту:

- **Тільки ваш chat_id.** Бот має слухати лише вас, інакше будь-хто отримає ваш сервер.
- **Заборона руйнівних дій без підтвердження** — пропишіть у system-промті (як вище) і/або перевіряйте команди на `rm -rf`, `reset`, `drop` перед виконанням.
- **Ліміт кроків** у циклі (20) — щоб агент не «закрутився».
- **Таймаут на bash** (напр. 45 с) — щоб одна команда не повісила бота.
- **Запасний простий режим** (`/simple`) — швидкий чат без дій на сервері, коли інструменти не потрібні.

---

## Як давати команди боту

- Формулюйте як задачу, а не питання: «створи скрипт, що…», «перевір…», «онови…».
- Складні речі бот розіб'є на кроки сам — не треба розписувати кожну команду.
- Хочете спершу побачити план — попросіть: «спочатку опиши, що зробиш, потім роби».
